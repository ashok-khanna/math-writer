Introduction

Thank you for visiting this page (Math Writer). Math Writer is a rich text web editor for mathematics, utilising the powerful MathJax library for rendering math and the TinyMCE library for a powerful, bug-free rich text experience. It is aimed at desktop users (not mobile) who want to generate HTML or pdfs of well formatted rich text and mathematics. Apart from its GUI interface, Math Writer supports Markdown for text entry, AsciiMath and LaTex for math entry. You can enter common mathematical symbols with keyboard shortcuts (refer shortcuts tab).

The Basics

Math Writer has four navigational tabs: Editor, Preview, Shortcuts and Read Me. You can navigate between these tabs through Ctrl + Shift 1, Ctrl + Shift + 2, Ctrl + Shift + 3 and Ctrl + Shift + 4 keyboard shortcuts respectively.
Tab Guide
Editor

The rich text editor is contained within the editor tab. You can enter can text, tables, lists, and of course, mathematical input (refer notes below). The editor allows you to format non-mathematical content through the various buttons for bold, italics, underline, color changes, text styles and so on. You can also use Markdown.

Math input on the other hand is not rendered in the editor tab - rather, you will see the raw code here, allowing you to modify it quickly. The preview tab renders mathematical content.

There are four very useful buttons in the editor toolbar:

  • Char Map (top left of the editor toolbar): Toggles the visibility of a set of comonly used symbols that you can insert into your file
  • Import Content (right end of the editor toolbar): Import html or text files stored on your computer into the editor (overriding anything that may currently be entered in the editor)
  • Export Content (right end of the editor toolbar): Export the raw HTML markup of the content you have entered (e.g. to save your work or to publish into your personal website). Refer notes below if you want to re-use this content in your personal webpages
  • <> (top right of the editor toolbar): Quickly see the raw HTML markup of your file, useful for finely editing the raw HTML markup in a way that is not possible when editing in the default formatted view. Hopefully you do not need to this ever, but the option is there
Preview The preview tab will render your math and display the final output. If you print this page to pdf (via your browser), this output will be exported to pdf.
Shortcuts The shortcuts tab contains a list of keyboard shortcuts supported by this app to enter mathematical symbols. You can modify these shortcuts by downloading a local copy of the repo and modifying the shortcuts in writer.js (modify the TinyMCE configuration variable).

Using Markdown

The rich text editor supports either formatting non-math input through the various GUI buttons in the editor's toolbar or through Markdown commands. Headers can be added via Markdown by starting a newline with #, ##, ... ###### (6 levels of heading with # representing H1). Italics can be entered by enclosing text with an asterisk *, whilst bold can be entered by enclosing text with a double asterisk **. You can start lists any of the following characters (the enclosing quotations are not to be included), each providing an alternate type of list (bullet, numerical, alphabetical, roman): "*", "-", "1.", "1)", "a.", "a)", "i." and "i)".

How to Enter Math

The editor supports both AsciiMath or LaTex input. Math can be entered in the editor either inline within a sentence of text, or by itself as a block of math. I prefer AsciiMath for inline math as it is easier to read, and LaTex for blocks as it has better equation alignment (but it is ultimately your preference). Below are illustrative examples.

Input Method Delimiter Input Example Output
Inline AsciiMath (recommended) Backticks ` ... ` `alpha ∩ beta = 3` `alpha ∩ beta = 3`
Inline LaTex (not recommended) Backslash then parentheses \( ... \) \(\alpha \cap \beta = 3\) \(\alpha \cap \beta = 3\)
Block AsciiMath without alignment (not recommended) Backticks ` ... ` `f(x) = x + 3 - 2`
`= x + 1`
`f'(x) = 1`		 `f(x) = x + 3 - 2`
`= x + 1`
`f'(x) = 1`
Block AsciiMath with alignment through matrices (not recommended) Backticks ` ... ` `{:(f(x) ,=, x + 3 - 2),( ,=, x + 1 ),(f'(x) ,=, 1 ):}` `{:(f(x) ,=, x + 3 - 2),( ,=, x + 1 ),(f'(x) ,=, 1 ):}`
Block LaTex (recommended) Backslash then square brackets \[ ... \] or double dollar signs $$ ... $$ \[\begin{aligned}f(x) &= x + 3 - 2 \\ &= x + 1\end{aligned}\] $$f'(x) = 1$$ \[\begin{aligned}f(x) &= x + 3 - 2 \\ &= x + 1\end{aligned}\] $$f'(x) = 1$$

Note that the last two examples for block AsciiMath and LaTex input appear convoluted. I suggest avoiding AsciiMath block input as it requires you to correctly place each part of the equation into a matrix: the {: ... :} are to start a matrix, the () are for each row and the , is used to split each row into its cells. On the other hand, whilst the block latex input looks complicated, that is mostly because I squashed it all onto one line (to illustrate that line breaks are ignored). It only has a few basic rules and is a straightforward method to input multi-line equations:

  • Equations must begin with \begin{aligned} and end with \end{aligned}
  • &= is used for aligning the equations
  • \\ is used to break to a new line

More details on the AsciiMath and LaTex commands to input math can be respectively found here and here.

Printing to Pdf

To print your content to pdf, first navigate to the Preview tab to render your Math. Once your Math is rendered, you can simply print to pdf using your browser's print functionality. Note that in your browser's print options, you can select to hide the header and footer that is automatically added by your browser to the print selection.

Exporting Math to the Web

You can export the raw HTML markup behind your content via a button in the editor. To subsequently include this on your website or blog, you will need to load the MathJax JavaScript library as well. Add the following code to your website:

<script>
window.MathJax = {
    loader: { load: ['[tex]/ams'] },
    tex: { packages: {'[+]': ['ams']} },
    "fast-preview": { disabled: true },
    extensions: ["tex2jax.js", "asciimath2jax.js", "TeX/AMSmath.js", "TeX/AMSsymbols.js", "TeX/noErrors.js"],
    jax: ["input/TeX", "input/AsciiMath", "output/HTML-CSS"],
    displayAlign: "left",
    tex2jax: {
      inlineMath: [ ["\\(","\\)"] ],
      displayMath: [ ['$$','$$'], ["\\[","\\]"] ],
      processEscapes: true
    }, 
    asciimath2jax: { delimiters: [['`','`']], preview: [""]},
    "HTML-CSS": { fonts: ["STIX"] }
  }
</script>
