Introduction

Thank you for visiting this page. This is a simple web-based editor for inputting mathematics, aimed for desktop users and not mobile. It utilises MathJax and the TinyMCE, allowing you to advanced mathematical typesetting with the comforts of a powerful, bug-free rich text editor. You can print your content (using your browsers print function) to pdf or alternatively export the HTML markup to re-use on your websites (refer ending notes below). It is the latter that is the primary motivation of this tool - to allow quick generation of mathematical content for the web.

The main benefit of this app is its preset list of keyboard shortcuts to allow you to quickly input commonly used mathematical symbols and the editor's ability to mix formatted text, tables, lists and so on with mathematical content. It's not a full editor, but covers enough functionality to create reasonable quality math output, in a simple, clean design. You may be able to extend this for more advanced use cases and this is encouraged. The code for this web app is well-documented on GitHub.

The Basics

Tab Tutorial
Editor

The editor tab is where you input your content, which can be text, tables, lists, and of course, mathematical input (refer notes below). The editor allows you to format non-mathematical content through the various buttons for bold, italics, underline, color changes, text styles and so on. Math input on the other hand is not rendered in the editor tab - you will see the raw code, allowing you to modify it quickly (the preview tab renders mathematical input).

There are three important buttons in the editor tab. Clicking the Char Map button on the editor allows you toggle the visiblity of a set of buttons for commonly used shortcuts. Clicking the export to HTML button allows you to copy the raw HTML markup of your content to your clipboard, allowing you to copy it into another application (refer notes below on how to create web pages from the HTML markup). Finally, the <> button allows you to quickly see the raw HTML markup, to make any edits that are not possible in the default formatted view.
Preview The preview tab will render your math and display the final output. If you print this page to pdf (via your browser), this output will be exported to pdf.
Shortcuts The shortcuts tab contains a list of keyboard shortcuts supported by this app to enter mathematical symbols. To modify and add to these shortcuts, you will need to fork this repo and update the TinyMCE configuration in writer.js.

How to Enter Math

The editor supports both AsciiMath or LaTex input. Math can be entered in the editor either inline within a sentence of text, or by itself as a block of math. I prefer AsciiMath for inline math as it is easier to read, and LaTex for blocks as it has better equation alignment (but it is ultimately your preference). Below are illustrative examples.

Inline LaTex (not recommended) Backslash then parentheses \( ... \) \(\alpha \cap \beta = 3\) \(\alpha \cap \beta = 3\)

Block AsciiMath with alignment through matrices (not recommended) Backticks ` ... ` `{:(f(x) ,=, x + 3 - 2),( ,=, x + 1 ),(f'(x) ,=, 1 ):}` `{:(f(x) ,=, x + 3 - 2),( ,=, x + 1 ),(f'(x) ,=, 1 ):}`
Block LaTex (recommended) Backslash then square brackets \[ ... \] or double dollar signs $$ ... $$ \[\begin{aligned}f(x) &= x + 3 - 2 \\ &= x + 1\end{aligned}\] $$f'(x) = 1$$ \[\begin{aligned}f(x) &= x + 3 - 2 \\ &= x + 1\end{aligned}\] $$f'(x) = 1$$

Note that the last two examples for block AsciiMath and LaTex input are complicated. I suggest avoiding AsciiMath block input as it requires you to correctly place each part of the equation into a matrix cell: the {: ... :} are to start a matrix, the () are for each row and the , is used to split each row into its cells. On the other hand, whilst the block latex input looks complicated, that is mostly because I squashed it all onto one line (to illustrate that line breaks are ignored). It only has a few basic rules and is a straightforward method to input multi-line equations:

  • Equations must begin with \begin{aligned} and end with \end{aligned}
  • &= is used for aligning the equations
  • \\ is used to break to a new line

More details on the AsciiMath and LaTex commands to input math can be respectively found here and here.

Exporting Math to the Web

You can export the raw HTML markup behind your content via a button in the editor. To subsequently include this on your website or blog, you will need to load the MathJax JavaScript library as well. Add the following code to your website:

<script>
window.MathJax = {
    "fast-preview": {
                disabled: true
        },
    extensions: ["tex2jax.js", "asciimath2jax.js"],
    jax: ["input/TeX", "input/AsciiMath", "output/HTML-CSS"],
    displayAlign: "left",
    tex2jax: {
      inlineMath: [ ["\\(","\\)"] ],
      displayMath: [ ['$$','$$'], ["\\[","\\]"] ],
      processEscapes: true
    }, 
    asciimath2jax: {
      delimiters: [['`','`']],
      preview: [""]},
      "HTML-CSS": { fonts: ["STIX"] }
  }
</script>
<script type="text/javascript" async src="https://cdnjs.cloudflare.com/ajax/libs/mathjax/2.7.7/MathJax.js"></script>
